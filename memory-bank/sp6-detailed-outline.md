# SP6 詳細大綱（Detail Outline）

## 專案定位
- 專案：自適應學習路徑整合專案（Adaptive Learning Path Integrator）
- 角色：AI-supported PM + Data Engineer + DB Manager（整合）
- 目標：整合 SP2-SP5 成端到端流程，提供可展示的個人化學習路徑。
- 最終輸出：整合 API、推薦規則、整合頁面、Demo 與回顧文件。

## 開發引導（先做後做）
1. 先對齊模組資料契約。
2. 再實作難度適配與推薦規則。
3. 再做前端整合頁面。
4. 最後完成 E2E 驗收與展示。

## 章節大綱

### Chapter 1 模組整合與資料流
- 做什麼：串接 SP2-SP5 輸入輸出。
- 怎麼做：統一欄位命名、建立 adapter、完成 smoke test。
- 產出：整合資料流文件與串接程式。
- 完成檢核：主流程可端到端執行。

### Chapter 2 難度適配規則
- 做什麼：根據學習表現輸出下一步建議。
- 怎麼做：定義分數區間、規則函式、邊界值測試。
- 產出：`recommendation_rules.py`。
- 完成檢核：不同輸入可得到穩定且合理的推薦。

### Chapter 3 前端整合頁面
- 做什麼：呈現進度、弱點、建議路徑。
- 怎麼做：整合 API 資料，做三區塊資訊展示與互動。
- 產出：整合 Dashboard 頁面。
- 完成檢核：使用者可理解目前狀態與下一步行動。

### Chapter 4 驗收與展示
- 做什麼：完成 Demo 與可重現文件。
- 怎麼做：設計 Demo script、補 README、跑 E2E 驗證。
- 產出：展示腳本、最終回顧（Retrospective）。
- 完成檢核：新成員依 README 可快速重現並展示。

## 作品集整理建議
- 展示 1：整合架構圖與資料流。
- 展示 2：推薦規則輸入輸出案例。
- 展示 3：端到端 Demo 錄影與回顧重點。
